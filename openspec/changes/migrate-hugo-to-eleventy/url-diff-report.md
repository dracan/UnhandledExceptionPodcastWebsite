## Accepted deltas (Hugo-only, not reproduced in Eleventy)

These outputs from Hugo are intentionally not reproduced and have been judged safe to drop. Each is either internal Hugo machinery with no public use, or a Hugo auto-generated artifact that isn't linked from the site navigation.

| Hugo path | Rationale |
|---|---|
| `/index.xml` | Hugo site-wide RSS. The public RSS is Buzzsprout (`feeds.buzzsprout.com/978640.rss`), already referenced from every page's `<head>`. No evidence the Hugo RSS was consumed externally. |
| `/tags/<t>/index.xml` (89 files) | Per-tag RSS auto-generated by Hugo. Not linked from the site. |
| `/posts/index.xml`, `/pages/index.xml` | Per-section RSS auto-generated by Hugo. Not linked from the site. |
| `/categories/index.html`, `/categories/index.xml` | Hugo default taxonomy page. This site uses `tags` only; no content sets `categories`. The category index has always been empty. |
| `/posts/index.html`, `/posts/page/{1..9}/index.html` | Hugo auto section listing for `posts/`. Not in the site menu (menu uses `/`, `/tags/`, `/pages/about/` etc.). Any visitor manually navigating to `/posts/` will now 404. Acceptable — the curated list is at `/`. |
| `/pages/index.html`, `/pages/page/1/index.html` | Hugo auto section listing for `pages/`. Not linked from the menu (menu links to individual pages like `/pages/about/`). |
| `/page/1/index.html` | Hugo duplicates page 1 of home at both `/` and `/page/1/`. Pagination UI links to `/page/2/` onwards, so `/page/1/` is never navigated to in normal use. |
| `/tags/<t>/page/1/index.html` | Same duplication pattern for tag pagination. Tag pagination UI never links here. |
| `/avatar.jpg` | m10c theme default avatar. The real site avatar is `/images/UepLogoText.png` and that's what `_data/site.js` references. No HTML emitted by either site references `/avatar.jpg`. |
| `/css/main.min.<hash>.css` | Hugo's fingerprinted SCSS output. Eleventy emits `/css/main.css`. The HTML `<link rel="stylesheet">` in `head.njk` references the Eleventy filename, so this rename is internal. External consumers don't cache static CSS by that specific URL. |

## Verification method

```bash
# Baseline: build Hugo at the last pre-migration commit in a worktree
git worktree add /tmp/ue-hugo HEAD
cd /tmp/ue-hugo && git submodule update --init && hugo --theme=m10c

# Candidate: build Eleventy
pnpm run build

# Sort and diff
(cd /tmp/ue-hugo/public && find . -type f | sort) > /tmp/hugo-paths.txt
(cd _site && find . -type f | sort) > /tmp/eleventy-paths.txt
diff /tmp/hugo-paths.txt /tmp/eleventy-paths.txt
```
